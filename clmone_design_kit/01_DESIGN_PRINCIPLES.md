# CLM One Design Principles

## 1. Work first, widgets second

CLM One is a legal workflow system. The primary visual object should usually be a work queue, contract record, form, workflow rail, risk list, document table, or audit log. Charts are only allowed when they clarify operational status.

## 2. Calm hierarchy

Use whitespace, alignment, typography, and subtle borders to create hierarchy. Do not rely on loud color blocks.

Page hierarchy:

1. App shell
2. Page title and primary action
3. Workflow/status context
4. Main work object
5. Supporting right rail
6. Secondary metadata

## 3. Enterprise density without crowding

The UI should not be sparse like a landing page. It should have real product density, but every region needs breathing room.

Preferred rhythm:

- 24px page gutters
- 20px card padding
- 16px field gaps
- 12px row gaps
- 40–44px input/button height

## 4. Status is quiet unless action is urgent

Use teal for good/progress/compliance/trusted states. Use muted gray for future/inactive states. Use burnt orange for creation and meaningful action. Use red only for true risk, breach, destructive action, or required field asterisk.

## 5. Legal users need confidence

Show routing, ownership, deadlines, risk, and audit information clearly. Avoid UI that hides accountability.

## 6. Every page should scale

Design patterns must work for Dashboard, New Contract, Contract Workspace, Repository, Privacy, Approvals, Audit Trail, DPA Reviews, and future CLM modules.

## Anti-patterns

Avoid:

- random KPI grids
- colorful dashboard tiles
- fake pie charts
- gradient headers
- large hero banners
- over-rounded bubbly UI
- huge icons
- drop shadows that make cards float like sticky notes
- too many competing CTAs
- tables with weak action affordance
