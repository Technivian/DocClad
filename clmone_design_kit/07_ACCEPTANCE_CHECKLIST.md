# Acceptance Checklist

## Global shell

- [ ] Sidebar is deep navy and consistent across pages.
- [ ] Active nav item has teal accent and tinted background.
- [ ] Top bar has search, firm selector, utilities, avatar, and primary action.
- [ ] Page background is off-white/cool light-gray.
- [ ] Cards are white with subtle border and minimal shadow.
- [ ] Typography is crisp and enterprise-grade.
- [ ] Icons are thin and consistent.

## Color discipline

- [ ] Teal is used for active/progress/trusted/compliance states.
- [ ] Burnt orange is used only for primary creation/action buttons.
- [ ] Red is used only for required asterisk, high risk, errors, destructive action.
- [ ] No rainbow card set.
- [ ] No loud gradients.

## Page hierarchy

- [ ] Title and subtitle are clear.
- [ ] Primary action is obvious but not loud.
- [ ] Main work object is visually dominant.
- [ ] Right rail supports and does not compete.
- [ ] Tables/forms are readable and aligned.

## CLM quality

- [ ] Page shows relevant contract/workflow/risk/audit context.
- [ ] Next action is clear.
- [ ] Ownership and status are visible where relevant.
- [ ] Deadlines and overdue states are visible where relevant.
- [ ] Missing metadata is visible where relevant.
- [ ] Audit/compliance screens feel evidence-driven.

## Anti-template check

- [ ] No decorative analytics.
- [ ] No generic KPI confetti.
- [ ] No fake charts.
- [ ] No marketing copy inside product screens.
- [ ] No unnecessary visual noise.

## Accessibility

- [ ] Keyboard focus states are visible.
- [ ] Color is not the only status indicator.
- [ ] Text contrast is sufficient.
- [ ] Buttons and inputs have accessible labels.
- [ ] Tables have semantic headers.
- [ ] Empty/error/loading states are meaningful.
