# Contract lifecycle terminology

CLM One separates three dimensions. Use the correct noun in product copy, APIs,
and tests.

| Say | Mean | Do not say |
|---|---|---|
| **In progress** | Record status `IN_PROGRESS` | “Draft” as the contract record status |
| **Drafting** | Workflow stage `DRAFTING` | “Draft status” for the pipeline position |
| **Draft** (document) | Document state `DRAFT` | Using it for record status or stage |
| **Active** | Record status `ACTIVE` (live agreement) | “Active” as a synonym for Executed stage |
| **Executed** (stage) | Workflow stage `EXECUTED` (just signed) | Equating stage Executed with record Active |
| **Executed** (document) | Document state `EXECUTED` | Using it as contract record status |
| **Archived** | Record status `ARCHIVED` | Archived as a workflow stage |

Authority: [PDR 0002](../pdr/0002-contract-stage-and-status.md).
