# DocClad UI Page Inventory

Phase: 1 - Inventory first  
Date: 2026-07-09  
Scope: visible, template-rendering routes in the Django app. API endpoints, webhooks, export/download endpoints, and post-only mutation/action endpoints are listed as non-visual exclusions at the end.

## Inventory Rules

- Workspace-mode containment must remain intact. `law_firm_ops` is the default operating mode; `in_house_clm` changes the Command Center, selected matter/risk surfaces, and CLM navigation only where the existing code gates it.
- Redesign work must preserve route names, permissions, business logic, query scoping, and tenant isolation.
- Required redesign patterns are preliminary labels for later phases. They are not implementation approval.
- Screenshot requirements use `empty`, `populated`, `error`, `law_firm_ops`, and `in_house_clm` as applicable.

## Non-Visual Exclusions

The following route families are not page redesign targets in Phase 1 because they return JSON, redirects, downloads, exports, webhooks, or mutations without a visible page template. They still need guardrail tests where applicable.

- Health/security infrastructure: `_health`, `csp-report`, `toggle-redesign`, debug/reload routes.
- SCIM/SAML protocol endpoints except SAML selector/login-visible errors: `/scim/v2/...`, SAML ACS/logout/metadata.
- API families under `/contracts/api/...`: contract APIs, Salesforce/Netsuite/webhook APIs, executive analytics APIs, document upload/extraction APIs, jobs, admin, onboarding, billing, compliance, approvals, clause analytics, playbooks, search/privacy/ops APIs.
- Export/download endpoints: document download, reports export, privacy evidence export, security/session/activity exports, DPA memo export.
- Mutation/action endpoints: mark notification read/all-read, organization switch/member/invitation actions, workflow step complete/update/reorder/publish/clone/restore, signature send/reminder/transition/packet actions, DPA analysis/status/note/link/generate memo, checklist toggle, deadline complete, DD item toggle, search preset save/delete, clause variant/playbook create.
- Billing checkout/portal/webhook routes.

## Current Shared Design Issues

- Templates mix page-local CSS, legacy utility classes, and newer `dc-*` / `arch-*` / cockpit classes.
- Several pages have strong local identities but not one canonical DocClad component system.
- Table/list pages do not consistently expose empty states, filter bars, and primary/secondary action hierarchy.
- Form pages do not consistently use a shared section/action/readiness/support-rail pattern.
- Workspace-mode containment is present but implemented per page; redesign must not loosen it.
- Several dashboard/report pages risk drifting into decorative analytics; final standard forbids loud KPI palettes and decorative charts.
- Specialized workflow cockpits have improved behavior but still need a shared cockpit system before rollout.
