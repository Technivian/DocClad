# Page Archetypes

## Dashboard

Header -> operational summary -> KPI strip -> 8/4 work queue and intelligence
rail. Empty workspaces preserve the KPI structure at zero and place activation
directly beneath it, so users learn the stable dashboard before data arrives.

## Repository

Views and filters -> page header -> compact summary -> search/sort toolbar ->
record table. Saved views may move below primary content on mobile. Preserve
horizontal comparison for dense contract records.

## Contract Workspace

Identity and lifecycle header -> primary document/work surface -> 4-column
context rail -> tabs for document, tasks, approvals, risks, obligations,
comments, and audit. The next valid action remains visible.

## Matter Workspace

Matter identity -> status and ownership -> related contracts/documents/tasks ->
timeline and financial context. Do not duplicate the contract workspace inside
a matter.

## AI Review

Source document -> detected issue -> approved/playbook position -> suggested
resolution -> evidence and impact -> governed action. AI explanation never
replaces the source clause or approval requirement.

## Settings

Use a stable section index and two-column settings sections. Separate identity,
security, workflows, integrations, and billing. Destructive administration is
isolated from routine configuration.

## List And Queue

Page header -> saved views/status tabs -> search/filter toolbar -> dense rows or
table -> pagination. Bulk actions appear only after selection and state exactly
which records they affect.
