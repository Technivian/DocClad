# Page Archetypes

## Authenticated application header

The Command Center header is the canonical authenticated page header. Page
title and one-line context belong in `authenticated_page_title` and
`authenticated_page_subtitle`; they must not be repeated as a second large
heading inside the work canvas. Page-level actions remain with the page
content so global search and account controls keep a stable position.

Primary workspaces use one main column with an optional 296px context rail on
the right. Do not insert a second navigation rail between the global sidebar
and the main workspace.

## Authentication

Single brand mark -> legal-operations context -> credential form -> enterprise
identity providers -> access and trust guidance. Desktop may use a restrained
context/form split; mobile collapses to the credential form without changing
authentication behavior or validation.

## Dashboard

Shared shell header -> feature summary and aligned action rail -> compact KPI
strip -> connected governance strip -> full-width recent work surface. Empty
workspaces use short setup guidance rather than tall zero-state cards.

This Command Center composition is the reference for page rhythm: 32px desktop
outer spacing, 24px major gaps, 16px compact grids, equal-height siblings, and
content-led height rather than decorative whitespace.

## Repository

Views and filters -> page header -> compact summary -> search/sort toolbar ->
record table. Saved views may move below primary content on mobile. Preserve
horizontal comparison for dense contract records.

## Contract Workspace

Identity and lifecycle header -> primary document/work surface -> 4-column
context rail -> tabs for document, tasks, approvals, risks, obligations,
comments, and audit. The next valid action remains visible.

## Matter Workspace

Matter identity -> status and ownership -> related contracts/documents/tasks ->
timeline and financial context. Do not duplicate the contract workspace inside
a matter.

## AI Review

Source document -> detected issue -> approved/playbook position -> suggested
resolution -> evidence and impact -> governed action. AI explanation never
replaces the source clause or approval requirement.

## Settings

Use a stable section index and two-column settings sections. Separate identity,
security, workflows, integrations, and billing. Destructive administration is
isolated from routine configuration.

## List And Queue

Page header -> saved views/status tabs -> search/filter toolbar -> dense rows or
table -> pagination. Bulk actions appear only after selection and state exactly
which records they affect.
